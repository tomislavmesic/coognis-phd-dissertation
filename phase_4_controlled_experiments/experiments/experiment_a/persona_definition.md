# Persona Definition Screen

## Introduction

When asking a support system for help, which type of answer would you prefer to receive?

Imagine you asked a support system how to solve a Wi-Fi connection problem.

Different people prefer different ways of receiving explanations.
Below are examples of how the same advice could be presented in different styles.

Example 1 - Structured step-by-step explanations
1. Check your username and password.
2. Reset your password if needed.
3. Reconnect to the Wi-Fi network.

Example 2 - Analytical explanations with reasoning
Wi-Fi problems usually happen because of incorrect credentials or saved network settings. Resetting the password and reconnecting often resolves the issue.

Example 3 - Supportive explanations with context
Connection problems can be frustrating. Let’s try a few simple things that usually fix this issue.

Example 4 - Short practical instructions
Reset your password and reconnect to the Wi-Fi

## Persona Selection Question

**Please choose the answer style you would personally prefer.**

| Response Option | Mapped Persona |
| --- | --- |
| Answer style similar to Example 1 | STJ |
| Answer style similar to Example 2 | NTP |
| Answer style similar to Example 3 | NFJ |
| Answer style similar to Example 4 | SFP |

## Persona Logic

The study operationalizes four communication-style personas:

| Persona | Communication Style | Experimental Role |
| --- | --- | --- |
| STJ | Structured, procedural, step-by-step communication | Represents users who prefer ordered instructions and clear procedures. |
| NTP | Analytical, explanatory, reasoning-oriented communication | Represents users who prefer causal explanation and conceptual rationale. |
| NFJ | Supportive, contextual, reassurance-oriented communication | Represents users who prefer empathetic guidance with situational context. |
| SFP | Concise, practical, action-oriented communication | Represents users who prefer short, direct instructions. |
