# Experiment A - Evaluation of AI-Based Support Responses

Experiment A evaluates whether psychologically adapted AI-generated support responses improve perceived clarity, satisfaction, trust, reuse intention, and response preference when compared with generic AI-generated support responses.

## Study Objective

The objective is to test whether communication-style adaptation improves user evaluation of support responses in university service scenarios. The experiment supports the dissertation's broader investigation of how artificial intelligence and psychological profiling can enhance internal knowledge discovery and user-facing support.

## Experimental Design

Participants first select a preferred answer style, which maps them to one of four communication personas: STJ, NTP, NFJ, or SFP. They then evaluate six university support scenarios. Each scenario contains one generic response and one persona-adapted response.

A Group A/B counterbalancing design controls for response-order effects:

| Group | Response A | Response B |
| --- | --- | --- |
| Group A | Generic response | Adapted response |
| Group B | Adapted response | Generic response |

Participants are not informed which response is generic or adapted.

## Persona Logic

The study operationalizes four communication-style personas:

| Persona | Communication Style | Experimental Role |
| --- | --- | --- |
| STJ | Structured, procedural, step-by-step communication | Represents users who prefer ordered instructions and clear procedures. |
| NTP | Analytical, explanatory, reasoning-oriented communication | Represents users who prefer causal explanation and conceptual rationale. |
| NFJ | Supportive, contextual, reassurance-oriented communication | Represents users who prefer empathetic guidance with situational context. |
| SFP | Concise, practical, action-oriented communication | Represents users who prefer short, direct instructions. |

## Scenario Structure

Each scenario includes a university support context, a user prompt, a generic response, and four persona-adapted responses. The six scenarios cover student exchange, tuition payment delay, Wi-Fi access, suspicious email handling, attendance policy, and elective course selection.

## Evaluation Workflow

For each scenario, participants evaluate Response A and Response B using five-point scales for clarity, satisfaction, trust, and likely reuse. They also select which response they would prefer to receive. A closing control question captures overall response-style preference.
